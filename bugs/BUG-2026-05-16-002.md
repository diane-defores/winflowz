## Summary

- Title: Keyboard 123 long press has no visible feedback while already in number mode
- Current status: `open`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-fix BUG-2026-05-16-002`

## Reproduction

- Preconditions:
  - Android real device.
  - WinFlowz keyboard active.
  - Keyboard is already on the `123` number page.
- Steps:
  1. Open any text field with WinFlowz keyboard.
  2. Switch to the `123` number page.
  3. Long press the `123` action.

## Expected Behavior

- The user receives visible feedback that the number bar was pinned or activated.
- Acceptable fixes from user: return to `ABC` so the pinned number bar is visible, or show the pinned number bar while staying on the `123` page.

## Observed Behavior

- Long press is technically accepted, but when already on the `123` page there is no obvious visible change.
- The user can think the long press did nothing.

## Evidence

- `shipflow_data/workflow/TEST_LOG.md` entry `2026-05-16 - Keyboard crash recovery Android real-device QA`.
- User report: "quand on est deja sur la page 123 du clavier et qu'on fait un long press sur 123, on devrait retourner sur ABC ... sinon ... la barre number pinned s affiche sur la page 123".
